# DB-Backup-Daemon

Фоновый сервис (воркер) на C#, который сам подключается к базам данных, делает дампы, пакует их в архивы с паролем, проверяет, что архив не битый, и удаляет старые бэкапы, чтобы не забивать диск. 

## 🛠️ .env
```
BACKUP_DB_PATH=database.db
```
## 🚀 Стек технологий

*   **Язык:** C# (.NET 8.0 / 9.0)
*   **Интерфейс (GUI):** WPF (Паттерн MVVM)
*   **Логи:** Serilog
*   **Локальная база:** SQLite
*   **Качество кода:** Линтер StyleCop + файл `.editorconfig`

## ⚙️ Что должно делать приложение (ТЗ)

1.  **Работать в фоне:** Сервис запускается и крутится в системе 24/7, просыпаясь по таймеру.
2.  **Всеядные бэкапы:** Код пишется через интерфейсы, чтобы через конфиг можно было настроить бэкап абсолютно любой СУБД.
3.  **Безопасность:** Все дампы обязательно шифруются паролем в ZIP-архив.
4.  **Контроль:** Сервис проверяет, что бэкап реально создался и успешно читается, прежде чем слать отчет.
5.  **Умное удаление (Ротация):** Жесткое правило хранения файлов:
    *   **7** ежедневных бэкапов
    *   **4** еженедельных бэкапа
    *   **1** ежемесячный бэкап
    *   Все, что старее этих лимитов, автоматически удаляется.

## 🛠️ Правила для команды

В проекте подключен строгий контроль качества кода через файл `.editorconfig`. Нарушения правил оформления, структуры и именования выводятся во вкладку **"Список ошибок"** как Предупреждения (Warnings).

### ⚡ Как вручную очистить файл перед коммитом:
Перед отправкой изменений в Git обязательно зайдите в свои файлы и выполните ручную очистку:

* **Для выравнивания отступов и фигурных скобок:** нажмите горячие клавиши `Ctrl + K, затем Ctrl + D`.
* **Для удаления мусорных и неиспользуемых библиотек (ошибка CS8019):** нажмите `Ctrl + K, затем Ctrl + E`.
* **Для исправления неправильных имен классов/переменных:** если линтер подчеркнул название, нажмите `Ctrl + .` на этом слове и выберите автоматическое безопасное переименование.

В общий репозиторий разрешено отправлять только код с нулевым количеством предупреждений!
