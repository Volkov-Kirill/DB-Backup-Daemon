# DB-Backup-Daemon

Фоновый сервис (воркер) на C#, который сам подключается к базам данных, делает дампы, пакует их в архивы с паролем, проверяет, что архив не битый, и удаляет старые бэкапы, чтобы не забивать диск. 


## 🚀 Стек технологий

*   **Язык:** C# (.NET 8.0 / 9.0)
*   **Тип проекта:** Worker Service
*   **Логи:** Serilog
*   **Локальная база:** SQLite
*   **Качество кода:** Линтер StyleCop + файл `.editorconfig`

## 👥 Кто за что отвечает

### 👑 Кирилл Волков (Team Lead / DevOps / QA / Logging)
*   **DevOps & Инфраструктура:** Отвечает за окружение, переменные среды, пути к файлам бэкапов и СУБД, а также за потенциальное развертывание сервиса.
*   **Логирование:** Настраивает `Serilog`.
*   **Качество и стиль кода:** Конфигурирует `.editorconfig` и линтеры, чтобы у всей команды был идеальный и чистый код.
*   **QA / Тестирование:** Пишет автотесты для проверки логики ротации и создания архивов, чтобы сервис не сломался в продакшене.

### 💻 Никита Козлов (Backend-разработчик)
*   **Ядро сервиса (Бэкенд):** Пишет архитектуру приложения, интерфейсы и код для фонового воркера.
*   **Работа с СУБД:** Пишет логику для универсального подключения к базам данных и вызова CLI-утилит для снятия дампов.
*   **Безопасность и архивация:** Кодит сжатие полученных дампов в ZIP-архивы и установку надежного пароля.
*   **Алгоритм ротации:** Реализует саму математику и логику очистки старых бэкапов (схема 7/4/1).

## ⚙️ Что должно делать приложение (ТЗ)

1.  **Работать в фоне:** Сервис запускается и крутится в системе 24/7, просыпаясь по таймеру.
2.  **Всеядные бэкапы:** Код пишется через интерфейсы, чтобы через конфиг можно было настроить бэкап абсолютно любой СУБД.
3.  **Безопасность:** Все дампы обязательно шифруются паролем в ZIP-архив.
4.  **Контроль:** Сервис проверяет, что бэкап реально создался и успешно читается, прежде чем слать отчет.
5.  **Умное удаление (Ротация):** Жесткое правило хранения файлов:
    *   **7** ежедневных бэкапов
    *   **4** еженедельных бэкапа
    *   **1** ежемесячный бэкап
    *   Все, что старее этих лимитов, автоматически удаляется.

## 🛠️ Правила для команды

У нас настроен жесткий линтинг. Перед тем как делать `git commit` и пушить код в репозиторий, обязательно пишем в терминале команду для автоматического выравнивания кода:

```bash
dotnet format
```
Если этого не сделать, код с кривыми отступами или неправильными именами переменных просто не соберется!
