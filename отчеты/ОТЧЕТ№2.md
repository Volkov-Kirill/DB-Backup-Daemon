# Отчет по Контрольной точке №2: Ядро и Инфраструктура

### 1. Текущий статус разработки ядра
* Полностью спроектирована и внедрена архитектура MVVM с выделенным Data-слоем на базе SQLite. Написаны и функционируют базовые механизмы интерфейса: универсальный класс команд для обработки кликов и класс для динамического обновления окон. Разработаны модули бэкенд-ядра для снятия дампов и парольной архивации, а также созданы новые визуальные компоненты управления бэкапами.

### 2. Интеграция стандартов обслуживания (из 1-й части практики)
* **Изоляция секретов:**  Имя базы данных и ключевые параметры вынесены из исходного кода в локальное окружение с помощью пакета DotNetEnv, а сам файл секретов скрыт от Git-репозитория.
.env
DB_NAME=test.db
* **Логирование:** Логгер Serilog инициализируется при старте приложения с помощью класса LogSystemInit, настраивая параллельную запись структурированных JSON-логов в файлы с ежедневной ротацией. 
* **Контейнеризация:** Не используем.
